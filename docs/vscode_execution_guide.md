# 🎯 VS Code 脚本执行解决方案

## 问题描述
当在 VS Code 中运行脚本时，会弹出 "运行 zsh 命令？" 对话框，需要手动点击 "允许"。

## 🔧 解决方案

### 方案1: 使用智能执行器 (推荐)
```bash
# 智能分析脚本安全性，提供5秒自动确认
智能执行 /path/to/your/script.sh
```

### 方案2: 使用VS Code助手
```bash
# 交互式脚本选择和执行
允许执行
# 或者
脚本助手
# 或者直接执行特定脚本
允许执行 /path/to/script.sh
```

### 方案3: 传统方法
- 在 VS Code 弹出对话框时点击 "允许"
- 等待5秒让自动确认系统处理

## 🚀 快速命令

- `脚本助手` - VS Code 脚本助手

```bash
# 执行 AI 助理演示

# 执行环境检查
执行脚本 ~/.ai-assistant/scripts/env_check.sh

# 交互式选择脚本
允许执行
```

## 🛡️ 安全特性

### 智能执行器包含：
1. **脚本安全分析** - 检查潜在风险
2. **权限评估** - 分析所需权限级别
3. **5秒自动确认** - 可手动中断
4. **执行监控** - 显示执行进度和结果

### 安全评分系统：
- 🟢 90-100分: 安全脚本
- 🟡 70-89分: 需要注意
- 🔴 <70分: 潜在风险

## 🎯 最佳实践

1. **优先使用智能执行器**
   ```bash
   智能执行 /path/to/script.sh
   ```

2. **对于频繁使用的脚本，创建别名**
   ```bash
   alias 我的脚本='智能执行 ~/my_scripts/useful.sh'
   ```

3. **VS Code 设置优化**
   - 在 VS Code 中信任工作区可减少确认次数
   - 使用终端直接执行避免 VS Code 对话框
